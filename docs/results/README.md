# Feature-Ergebnisse

Für nutzerwirksame Features entsteht nach Pilot oder Go-live eine Datei `BV-NNN.md`.

```markdown
# Result — BV-NNN Feature-Name

**Live seit:** YYYY-MM-DD
**Pilot-Nutzer/Rolle:** ...
**Status:** holding | iterating | killed | shipped-as-is

## Hypothese
...

## Tatsächlich umgesetzt
...

## Outcome — Daten
- Nutzung:
- Fehler/Korrekturen:
- relevante Betriebsmetrik:

## Outcome — Feedback
- Beobachtung oder kurzes, freigegebenes Zitat:

## Überraschungen
...

## Lernen und nächste Iteration
- bestätigt:
- widerlegt:
- nächster Schritt:

## Referenzen
- Backlog: BV-NNN
- Commit/Release:
- Meeting/Entscheidung:
```

Keine unnötigen Gast- oder Mitarbeiterdaten in Ergebnisdateien aufnehmen.
